# LogiRisk

**Аналитическая система прогнозирования рисков инцидентов корпоративного автопарка.**
Проект разработан в рамках трека «Большие данные» Samsung Innovation Campus.

## 🛠 Стек технологий

* **ETL & Data Engineering:** Apache Spark (PySpark), Parquet
* **Machine Learning:** CatBoost, XGBoost, ExtraTrees, Logistic Regression
* **Оркестрация & Инфраструктура:** Apache Airflow, Docker Compose
* **UI-Дашборд:** Streamlit

## 🧠 Под капотом: Tiered Cascade Ensemble

Пайплайн (DAG) в Airflow автоматически собирает кросс-доменные фичи и исторические окна на PySpark, после чего передает данные в ML-ансамбль. Система использует двухуровневую архитектуру:

1. **Tier 1 (Критические):** CatBoost отбирает Топ-20 самых опасных водителей. Их риск масштабируется в диапазоне **[50–100]** для максимальной детализации.
2. **Tier 2 (Остальные):** Взвешенная сумма прогнозов всех моделей нормируется через Min-Max масштабирование в диапазон **[0–49.9]**.
3. **Safety Trigger (Предохранитель):** Если любая из 4 моделей фиксирует вероятность риска >90%, водитель принудительно уходит в «Критический» статус, чтобы сигнал не затерялся при нормализации.

## 🚀 Быстрый запуск (Docker)

Убедитесь, что порты `8501` и `8080` свободны.

**1. Подъем инфраструктуры:**

```bash
docker-compose up --build -d

```
**2. Запуск аналитики:**

* Откройте Airflow: `http://localhost:8080` (логин/пароль: `admin`).
* Запустите DAG `logirisk_analytical_pipeline`.
* После успешного выполнения (Spark Feature Engineering -> Train/Inference), результаты автоматически появятся на главной странице дашборда Streamlit.


**3. Дашборд и загрузка данных (При необходимости):**

* Перейдите в дашборд: `http://localhost:8501`
* На **Вкладке 3** загрузите новые телеметрические данные (CSV). Система автоматически выполнит безопасный **UPSERT** в Parquet-архив.
